# 指纹防护 (Fingerprint Defender)

一个基于确定性噪声的浏览器指纹防护扩展，保护您的隐私免受跨站追踪。

## 什么是浏览器指纹？

浏览器指纹是一种不依赖 Cookie 的追踪技术。它通过收集浏览器、系统和硬件信息来创建唯一的"指纹"，可以跨网站识别用户。本扩展通过添加确定性噪声，让追踪者无法建立准确的设备档案。

## 核心特性

### 🔒 确定性噪声技术

与简单的随机化不同，本扩展使用**基于域名的确定性噪声**：

- **同站稳定**：在同一网站内，您的指纹保持一致，避免被检测为反指纹工具
- **跨站隔离**：不同网站看到不同的指纹，防止跨站追踪关联
- **长期持久**：种子存储在扩展本地，关闭浏览器后指纹依然稳定
- **不可检测**：从网站角度看，您的浏览器有正常、稳定的指纹（只是与真实指纹略有差异）

### 🎯 防护范围

可独立控制以下指纹防护功能：

- **Canvas** – 向 Canvas 渲染添加噪声，防止图形指纹识别
- **字体** – 向字体测量值添加噪声，防止字体指纹识别
- **音频上下文** – 向 Web Audio API 输出添加噪声，防止音频指纹识别
- **WebGL** – 伪装 WebGL 参数，防止显卡指纹识别
- **WebGPU** – 向 WebGPU 设备信息添加噪声，防止硬件指纹识别
- **客户端矩形** – 向 DOM 元素测量添加噪声，防止布局指纹识别

## 工作原理

### 技术架构

```
首次访问网站
  ↓
为该域名生成随机种子（存储在扩展本地）
  ↓
基于 seed + 参数名称 生成确定性噪声
  ↓
应用噪声到指纹 API
  ↓
网站看到：稳定但略有偏差的指纹
```

### 实现细节

1. **种子管理**：
   - 使用 `chrome.storage.local` 为每个域名存储独立的随机种子
   - 种子在浏览器重启后保持不变，确保指纹稳定性

2. **噪声生成**：
   - 使用 FNV-1a 哈希算法：`noise = hash(seed + label)`
   - 同一参数在同一域名上总是返回相同的噪声
   - 避免调用顺序、时序问题导致的不一致

3. **代码注入**：
   - **ISOLATED world**：管理设置和种子，安全访问扩展 API
   - **MAIN world**：拦截指纹 API，应用确定性噪声

## 安装方法

### 作为未打包扩展加载 (Chrome/Edge/Brave)

1. **下载或克隆本仓库**到您的计算机

2. **打开扩展管理页面**：访问 `chrome://extensions` 或 `edge://extensions`

3. **启用开发者模式**（右上角开关）

4. **点击"加载已解压的扩展程序"**，选择本项目的文件夹

5. **扩展将出现在扩展列表中**，可以固定到工具栏方便访问

## 使用说明

### 基本操作

1. **点击扩展图标**打开控制面板

2. **使用开关**独立控制每个防护功能：
   - 默认配置：Canvas、音频上下文、WebGL 开启
   - 其他功能根据需要开启（可能影响兼容性）

3. **查看功能说明**：将鼠标悬停在功能名称旁的 `i` 图标上

4. **重置默认**：一键恢复到推荐的默认配置

5. **测试防护**：打开测试页面验证防护是否生效

### 刷新指纹

如果需要为某个网站生成新的指纹：

**方法一：右键菜单（推荐）**

- 在工具栏上**右键点击扩展图标**
- 选择"刷新指纹（需手动刷新页面）"
- 手动刷新页面（F5 或 Ctrl+R）

**方法二：控制面板**

- 点击扩展图标打开面板
- 点击"刷新指纹"按钮
- 手动刷新页面

> **注意**：删除种子后需要手动刷新页面，新指纹才会生效

## ⚠️ 故障排除

### 网站出现问题、加载缓慢或显示错误

某些网站可能不兼容指纹防护。尝试：

1. 关闭特定防护功能，逐个排查
2. 使用"重置默认"恢复推荐配置
3. 完全关闭所有防护（如果网站完全无法使用）

**常见问题来源**：3D 游戏、视频平台、实时应用、验证码服务

### 需要彻底清除所有数据

如果需要重置所有域名的种子：

1. 打开浏览器扩展管理页面
2. 找到"指纹防护"扩展
3. 点击"删除"然后重新安装

或者：

1. 打开开发者工具（F12）
2. 进入 **Application** → **Storage** → **Extensions** → **Local Storage**
3. 删除 `domainSeeds` 数据

## 技术说明

### 为什么使用确定性噪声而不是纯随机？

| 方案           | 同站稳定性  | 可检测性    | 跨站隔离    |
| -------------- | ----------- | ----------- | ----------- |
| **纯随机**     | ❌ 每次都变 | 🚫 易被检测 | ✅ 完全隔离 |
| **确定性噪声** | ✅ 完全稳定 | ✅ 难以检测 | ✅ 完全隔离 |

纯随机会导致：

- 指纹每次页面加载都变化 → 网站识别为"不正常行为"
- 触发反作弊系统、账号验证、功能限制

确定性噪声：

- 网站看到的指纹稳定、正常（只是略有偏差）
- 不会触发"指纹抖动"检测
- 实现真正的隐身防护

### 为什么按域名存储种子？

如果所有网站使用同一个种子：

- 所有网站看到相同的噪声模式
- 追踪者可以通过噪声特征关联跨站行为
- 失去跨站隔离保护

按域名存储：

- 每个域名独立的随机种子
- 不同网站无法关联指纹
- 实现真正的跨站隔离

### 与其他方案对比

**方案 A：基于域名哈希生成种子**

```javascript
seed = hash(domain); // 所有用户在同一网站的种子相同
```

- ❌ 大量用户的指纹噪声模式相同
- ❌ 可被识别为"使用某个特定扩展的用户群体"

**方案 B：随机生成 + 扩展存储**（本扩展采用）

```javascript
seed = random(); // 每个用户每个域名独立随机
storage.set(domain, seed);
```

- ✅ 每个用户的种子完全独立
- ✅ 无法通过噪声模式识别扩展
- ✅ 最强的隐私保护

## 隐私承诺

本扩展**绝不**：

- 收集您的数据
- 向外部服务器发送信息
- 追踪您的浏览行为
- 使用遥测或分析

所有防护均在浏览器本地完成，种子存储在扩展的私有存储空间，网站无法访问。
